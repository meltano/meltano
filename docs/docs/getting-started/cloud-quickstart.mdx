Get from zero to a running pipeline in five steps. No infrastructure setup required.

## Steps

<div className="gs-steps">
  <div className="gs-step">
    <div className="gs-step__num">1</div>
    <div className="gs-step__body">
      <div className="gs-step__title">Get in touch</div>
      <p className="gs-step__desc">Create your Meltano Cloud account at <a href="https://meltano.com/contact" target="_blank" rel="noopener noreferrer">meltano.com/contact</a>. You can sign in with GitHub or Google.</p>
    </div>
  </div>
  <div className="gs-step">
    <div className="gs-step__num">2</div>
    <div className="gs-step__body">
      <div className="gs-step__title">Create a workspace</div>
      <p className="gs-step__desc">A workspace is an isolated environment for your team. Name it after your organization or project, then invite teammates from the Members tab.</p>
    </div>
  </div>
  <div className="gs-step">
    <div className="gs-step__num">3</div>
    <div className="gs-step__body">
      <div className="gs-step__title">Connect a Git repository</div>
      <p className="gs-step__desc">Link the GitHub repo that contains (or will contain) your <code>meltano.yml</code>. Meltano Cloud reads your pipeline definitions directly from your repo on every run.</p>
    </div>
  </div>
  <div className="gs-step">
    <div className="gs-step__num">4</div>
    <div className="gs-step__body">
      <div className="gs-step__title">Add secrets</div>
      <p className="gs-step__desc">Store connector credentials in the Secrets vault. They are injected at runtime and never written to disk or logs.</p>
    </div>
  </div>
  <div className="gs-step">
    <div className="gs-step__num">5</div>
    <div className="gs-step__body">
      <div className="gs-step__title">Trigger a pipeline run</div>
      <p className="gs-step__desc">Use the Cloud UI or the <code>meltano cloud run</code> CLI command to kick off your first pipeline. Watch live logs stream directly in the interface.</p>
    </div>
  </div>
</div>

:::tip You're live
Once your first run succeeds you can set a schedule, configure external log streaming (DataDog), and add alerting — all from the Cloud UI.
:::

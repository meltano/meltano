# MySQL - MariaDB

## Overview

MySQL and MariaDB are popular open-source relational database management systems that enable users to store, organize, and retrieve data efficiently. They are widely deployed in web applications and offer transaction support, replication, and clustering capabilities suitable for projects ranging from small personal implementations to large enterprise deployments.

## Prerequisites

**Create a Database User:**
1. Create a dedicated user for the tap and grant the minimum permissions required for your chosen replication method.
2. For FULL_TABLE and INCREMENTAL replication, grant SELECT access to the target schemas.
3. For LOG_BASED replication, grant additional replication privileges at the global level: REPLICATION SLAVE and REPLICATION CLIENT permissions.

**Enable Binary Logging (LOG_BASED replication only):**
1. Add the following to your MySQL/MariaDB server configuration file: `log_bin = mysql-bin`, `binlog_format = ROW`, and `expire_logs_days = 7`, then restart the server.
2. For AWS RDS/Aurora: Set the `binlog_format` parameter to ROW and enable automated backups.
3. For Google Cloud SQL: Enable "Point-in-time recovery" and configure `binlog_format = ROW`.
4. For Azure Database for MySQL: Enable "Backup retention" and set `binlog_row_image = FULL`.

**Co-locate Your Pipeline:**
1. Deploy your pipeline in the same cloud region as your MySQL server to eliminate cross-region egress latency and bandwidth costs.
2. For on-premises databases, use the SSH tunnel settings to enable secure connectivity without exposing a public port.

## Configuration

| Setting | Description | Type |
|---------|-------------|------|
| **Host** | The hostname or IP address of the MySQL/MariaDB server | String |
| **Port** | The port number for the connection | Integer |
| **User** | The username for authentication | String |
| **Password** | The password for authentication | Credential |
| **Database** | The name of the database to connect to | String |
| **Cursor Class** | The class to use for the cursor | String |
| **Server ID** | The server ID to use for replication | Integer |
| **Filter DBs** | A list of databases to include or exclude from replication | Array |
| **Use GTID** | Whether to use global transaction identifiers for replication | Boolean |
| **Engine** | The storage engine to use for the connection | String |
| **SSL** | Whether to use SSL encryption for the connection | Boolean |
| **SSL CA** | The path to the SSL CA certificate file | String |
| **SSL Certificate** | The path to the SSL certificate file | String |
| **SSL Key** | The path to the SSL key file | String |
| **Internal Hostname** | The internal hostname to use for the connection | String |
| **Session SQLs** | SQL statements to execute when the connection is established | Array |
| **SSH Tunnel Host** | Address of the bastion host for SSH connection | String |
| **SSH Tunnel Port** | Port to connect to bastion host | Integer |
| **SSH Tunnel Private Key** | Base64 encoded private key for key pair authentication | Credential |
| **SSH Tunnel Private Key Password** | Private key password (optional) | Credential |
| **SSH Tunnel Username** | Username to connect to bastion host | String |
| **SSH Tunnel Password** | Password for basic authentication to bastion host | Credential |
| **Batch Size Rows** | Number of rows per BATCH message file; set to 0 for RECORD mode | Integer |
